# บันทึกการเปลี่ยนแปลง (Changelog) - Asthma-Flow

รายการบันทึกการพัฒนาและการเปลี่ยนแปลงทั้งหมดของระบบ **Asthma-Flow** ตั้งแต่เริ่มต้นจนถึงปัจจุบัน (เวอร์ชันปัจจุบัน: **v1.5.0**)

---

## [v1.5.0] - 2026-06-15
### Added (เพิ่มฟีเจอร์ใหม่)
- **ระบบจัดเก็บประวัติยาด้วยรหัสยา (Persistent Medication ID System):**
  - เพิ่มการบันทึกฟิลด์รหัสยา `c1_med_id`, `c2_med_id` และ `reliever_med_id` ลงในตาราง `medications` เพื่อเชื่อมโยงกับตาราง `medication_list` (Master) ด้วย Foreign Key คีย์รหัสยา
  - ปรับปรุง API `getLatestMedication` และ `getMedicationByDate` ในระดับ Data Access Layer (`lib/db.ts`) ให้ทำรายการ Join ดึงชื่อยาที่เป็นปัจจุบันเสมอจากตารางหลัก พร้อมคงระบบ fallback รองรับข้อมูลชื่อยาแบบเก่า (Backward Compatibility)
  - ปรับปรุงการบันทึกการตรวจรักษาในหน้า Visit Form (`app/staff/visit/[hn]/page.tsx`) ให้ค้นหารหัสยาจากตัวเลือกปัจจุบันและแนบรหัสยา (Medication IDs) ไปพร้อมกับการบันทึกประวัติการตรวจ
  - เพิ่มคอลัมน์รหัสยาทั้ง 3 ฟิลด์เข้าสู่ระบบสำรองข้อมูลอัตโนมัติประจำวันไปยัง Google Sheets (`app/api/backup/route.ts`)
- **ระบบกรองสถิติและเปรียบเทียบแนวโน้มรายปีงบประมาณ (Thai Fiscal Year & Quarter Filters):**
  - เพิ่มมุมมองตัวเลือก **"รายปีงบประมาณ" (Fiscal Year)** ในระบบกรองสถิติหน้า Dashboard เพื่อคำนวณและแสดงผลข้อมูลตามปีงบประมาณไทย (1 ตุลาคม - 30 กันยายน)
  - ปรับระบบคำนวณไตรมาสให้เป็นปีงบประมาณ (ไตรมาส 1: ต.ค.-ธ.ค., ไตรมาส 2: ม.ค.-มี.ค., ไตรมาส 3: เม.ย.-มิ.ย., ไตรมาส 4: ก.ค.-ก.ย.)
  - เพิ่มระบบแสดงแนวโน้มสถิติย้อนหลัง 5 ปีงบประมาณในกราฟ composed chart เพื่อการวิเคราะห์แนวโน้มสุขภาพและปริมาณผู้ป่วยระยะยาว
- **ระบบจัดเรียงตารางผู้ป่วยนัด (Appointments Roster Sorting):**
  - เพิ่มแถบเครื่องมือจัดเรียงข้อมูลรายชื่อผู้ป่วยนัดประจำวันอังคาร โดยรองรับการจัดเรียงตามเลข HN และตามตัวอักษรชื่อจริง (ไม่นับรวมคำนำหน้านาม เช่น นาย, นาง, น.ส., ด.ช., ด.ญ. ฯลฯ)
  - ออกแบบปุ่มสลับการจัดเรียงในสไตล์ Warm Retro Neo-brutalism ที่สวยงามและตอบสนองการคลิกทันที

### Changed (การเปลี่ยนแปลง)

  - ปรับเปลี่ยนดีไซน์ธีมบัตรใหม่ โดยตัดธีมสี Luxury Charcoal ออกตามความต้องการ และแทนที่ด้วยดีไซน์ใหม่สไตล์ Warm Orange ที่มีเฉดสีอบอุ่นและเป็นทางการ
  - ปรับระบบสีและการแสดงผลของแต่ละธีม (Clinical Blue, Clinical Green, Clinical Purple, และ Warm Orange) ให้รองรับหน้าบัตรแบบ High Fidelity ที่ดึงสไตล์จากภาพต้นแบบมาใช้งานได้อย่างสมบูรณ์

## [v1.4.6] - 2026-05-26
### Added (เพิ่มฟีเจอร์ใหม่)
- **ระบบจัดทำและพิมพ์บัตรประจำตัวผู้ป่วยแบบกลุ่ม (Printable Asthma Flow Alert Cards Portal):**
  - พัฒนาพอร์ทัลอินเตอร์แอคทีฟเพื่อให้เจ้าหน้าที่สามารถเลือกวันที่มีนัดหมายเพื่อจัดทำและพิมพ์บัตรประจำตัวผู้ป่วย (Asthma Flow Alert Card) ขนาดมาตรฐานบัตร 8.6 x 5.4 ซม. บนกระดาษ A4
  - ระบบรองรับการค้นหาผู้ป่วย เลือกผู้ป่วยทั้งหมด/บางส่วน เปิด/ปิดการแสดงผลเส้นประสำหรับการตัด และการสลับธีมสีบัตรได้อิสระ
  - รองรับการเลือกธีมสีพิเศษ 4 รูปแบบหรูหรา ได้แก่ Clinical Blue, Clinical Green, Clinical Purple, และ Luxury Charcoal
  - ปรับข้อมูลบนหน้าบัตรประจำตัวให้คลีนมินิมอลตามความเหมาะสมทางการแพทย์ โดยจะแสดงผลเฉพาะ: ชื่อ-สกุล, เลข HN, และรหัส QR Code สาธารณะสำหรับตรวจประวัติการรักษา
  - บังคับใช้ฟอนต์ **Sarabun** 100% ทั้งในหน้าจอพรีวิวก่อนพิมพ์และเมื่อส่งพิมพ์ผ่านเครื่องพิมพ์ (Print Media Direct Override) เพื่อความสวยงามเป็นทางการ
  - ออกแบบเลย์เอาต์จัดเรียงหน้ากระดาษ A4 อิงสัดส่วนตามขนาดขอบกระดาษจริง เพื่อให้จัดวางพิมพ์การ์ดพอดี 10 ใบต่อ 1 หน้ากระดาษ (2 คอลัมน์ x 5 แถว) ไร้ปัญหาหน้าเลื่อนหรือขอบขาด

## [v1.4.5] - 2026-05-25
### Fixed (แก้ไขข้อผิดพลาด)
- **ระบบสถิติหน้า Dashboard (New Patient Count):** แก้ไขการคำนวณจำนวนผู้ป่วยใหม่หน้า Dashboard ของเจ้าหน้าที่ให้ใช้แฟล็ก `is_new_case` จากตาราง `visits` ในฐานข้อมูล Supabase แทนการคำนวณแบบเดิมที่ผิดพลาด ทำให้สามารถแสดงจำนวนผู้ป่วยใหม่ได้อย่างถูกต้องแม่นยำ
- **ระบบบันทึก Daily PEFR ของพยาบาล:** ปรับปรุงการแสดงผลค่าเปอร์เซ็นต์ predicted PEFR ในตารางบันทึกประจำวันของพยาบาล ให้แสดงผลในรูปแบบทศนิยม 2 ตำแหน่ง (`%.2f`) เพื่อความละเอียดและมาตรฐานในการประเมินทางการแพทย์
- **การจัดการความขัดแย้งของข้อมูลในฐานข้อมูล (Database Graceful Edit Handling):** เพิ่มการทำงานในระดับ Data Access Layer (ใน `lib/db.ts`) ให้รองรับการแก้ไขข้อมูลประวัติการรักษา (Edit Visit) ในกรณีที่มีรายการข้อมูลยา (Medications) หรือการประเมินเทคนิคการพ่นยา (Technique Checks) ซ้ำซ้อน หรือสูญหายอย่างสง่างาม (Graceful Handling) โดยไม่ทำให้เกิดข้อผิดพลาดในการรันแอปพลิเคชันหรือบิลด์ล้มเหลว

  - แสดงผลรายการผู้ป่วยที่นัดหมายมาตรวจในวันปัจจุบัน พร้อมการคำนวณและประเมินค่าทำนายระดับ PEFR (Predicted PEFR) และเทียบเป็นเปอร์เซ็นต์ตามมาตรฐานทางการแพทย์ให้อัตโนมัติทันทีที่บันทึก
  - พัฒนาระบบค้นหาผู้ป่วยนอกตารางนัดหมาย (Walk-in Patients Search) เพื่อเพิ่มรายชื่อบันทึกของวันปัจจุบันได้ พร้อมป้องกันการสร้างประวัติซ้ำซ้อนในวันเดียวกันอย่างรัดกุม

## [v1.1.0] - 2026-02-15
### Added (เพิ่มฟีเจอร์ใหม่)
- **ระบบคัดกรอง Drug-Related Problems (DRP System):**
  - พัฒนาระบบประเมินปัญหาจากการใช้ยา (DRP) ประกอบด้วยการระบุหมวดหมู่ปัญหา (Category), ชนิดปัญหา (Type), สาเหตุ (Cause), แผนการแก้ไขของเภสัชกร (Intervention) และผลลัพธ์การแก้ไข (Outcome)

  - เพิ่มระบบตรวจสอบ "1-day-1-visit" จำกัดการบันทึกประวัติการรักษาผู้ป่วยได้ไม่เกิน 1 ครั้งต่อวันต่อคน เพื่อป้องกันการกรอกข้อมูลซ้ำซ้อน
  - ระบบลบข้อมูลระดับแอดมิน (Admin Deletion System) สามารถลบประวัติการรักษาหรือลบตัวผู้ป่วยแบบถาวร โดยระบบจะลบข้อมูลที่เชื่อมโยงกันแบบลดหลั่นลงไป (Cascading Delete) โดยอัตโนมัติ
  - เพิ่มฟังก์ชันคัดเรียงข้อมูลรายชื่อผู้ป่วย (Sorting) ตามลำดับวันที่นัดตรวจครั้งถัดไป ตามด้วยการเรียงตามเลข HN ของผู้ป่วย เพื่อความสะดวกรวดเร็วในการทำงาน
- **PDPA Compliance:**
  - เพิ่มระบบความยินยอมการเปิดเผยข้อมูลส่วนบุคคล (PDPA Consent Prompt & Cookies)
  - ปิดบังการเข้าถึงประวัติโดยตรงผ่าน URL และเข้ารหัสลิงก์สำหรับการแสดงผลแผนการรักษาสำหรับผู้ป่วย
